База данных (БД) — это организованная структура, предназначенная для хранения, изменения и обработки взаимосвязанной информации,
преимущественно больших объемов.

Система управления базами данных (СУБД) —  это совокупность языковых и программных средств, которая осуществляет доступ к данным,
позволяет их создавать, менять и удалять, обеспечивает безопасность данных и т.д.

Базы данных NoSQL (нереляционная бд) - в базах данных NoSQL для доступа к данным и управления ими применяются различные модели данных,
в том числе документная, графовая, поисковая, с использованием пар «ключ‑значение» и хранением данных в памяти.
Базы данных таких типов оптимизированы для приложений, которые работают с большим объемом данных,
нуждаются в низкой задержке и гибких моделях данных.

Реляционная база данных – это набор данных с предопределенными связями между ними. Эти данные организованны в виде набора таблиц,
состоящих из столбцов и строк. В таблицах хранится информация об объектах, представленных в базе данных.
В каждом столбце таблицы хранится определенный тип данных, в каждой ячейке – значение атрибута.
Каждая стока таблицы представляет собой набор связанных значений, относящихся к одному объекту или сущности.
Каждая строка в таблице может быть помечена уникальным идентификатором,
называемым первичным ключом, а строки из нескольких таблиц могут быть связаны с помощью внешних ключей.

SQL - язык структурированных запросов, основной задачей которого является предоставление простого способа
считывания и записи информации в реляционнцую базу данных.


В MYSQL могут быть созданы следующие объекты:
1. Таблицы (table). В таблицах хранятся данные

2. Представления (view). Представление доступно для пользователя как таблица, но само оно не содержит данных,
а извлекает их из таблиц в момент обращения к нему.
Если данные изменены в базовой таблице, то пользователь получит актуальные данные при обращении к представлению,
использующему данную таблицу

3. Индексы (index) применяются для быстрого поиска строк с указанным значением одного столбца.
Индексы используются для того, чтобы:
Быстро найти строки, соответствующие выражению WHERE.
Извлечь строки из других таблиц при выполнении объединений.
Найти величины MAX() или MIN() для заданного индексированного столбца и тд.

4. Хранимые процедуры (procedure) -  набор SQL команд, представляющие собой программную логику сохраняемую в БД,
которую в последствии можно будет вызвать. При вызове процедуры можно передать параметры.
Процедура не возвращает значение, как функция.
хранимые функции (function) - набор SQL команд, представляющие собой программную логику сохраняемую в БД,
которую в последствии можно будет вызвать. При вызове процедуры можно передать параметры. Функции возвращают значения.

5. Триггеры (trigger) - хранимые процедуры, которые срабатывают автоматически при наступлении определенных событий
при работе с данными (добавление / изменение / удаление).

Синтаксис sql делят на:

1. DDL - определение данных
	1) создать БД
	2) удалить БД
	3) создать таблицу
	4) удалить таблицу
	5) создать первичный/внешний ключ и тд

2. DML - манипулирование данными
	1) добавление записей в таблицу
	2) получение данных в таблице
	3) удаление данных из таблицы
	4) обновление данных в таблице

3. DCL - управление
	1) управление правами доступа
	2) управление транзакциями
	и тд


Связи между таблицами:
1. Один к одному - одна запись в одной таблице соответствует только
одной записи в другой.
2. Один ко многим - одна запись в одной таблице соответствует
нескольким записям в другой. Реализуется как дополнительный столбец
3. Многие ко многим - несколько записей в одной таблице соответствуют
нескольким записям в другой. Реализуется как дополнительная таблица.
